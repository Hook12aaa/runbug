# AX Protocol — Runbug Wire Format

All events on every runbug channel (console-forward, AX-snapshot, command-channel) are NDJSON — one JSON object per line. The shim, the backend endpoint, and any future consumer implement exactly this schema.

## Event envelope

Every event has:

| Field | Type | Required | Description |
|---|---|---|---|
| `type` | string | yes | One of: `console`, `snapshot`, `action`, `action-result`, `shim-ready`, `configure`, `dom-event` |
| `ts` | string | yes | ISO 8601 UTC timestamp, millisecond precision (e.g., `2026-04-22T12:34:56.789Z`) |
| `tabId` | string | no | Per-shim UUID generated at mount. Present on every shim-emitted event in v1.3+; absent on backend-emitted events unless echoed. |

Additional fields per type below.

## type: console

Emitted by the shim when the target page calls `console.*`.

| Field | Type | Description |
|---|---|---|
| `level` | string | `log`, `info`, `warn`, `error`, `debug` |
| `args` | array | console.* arguments, passed through `safeSerialize` (strings, numbers, booleans, null, plain objects, arrays — functions and DOM nodes stringified to `[Function]` / `[Node: <tagName>]`) |

Example:
`{"type":"console","ts":"2026-04-22T12:34:56.789Z","level":"error","args":["WebGL: INVALID_OPERATION: bindTexture"]}`

## type: snapshot

Emitted by the shim in response to a command-channel `snapshot` request. Contains the current page's AX tree, pruned to interactive and landmark elements only.

| Field | Type | Description |
|---|---|---|
| `tree` | array | Ordered list of AX nodes (see schema below) |
| `url` | string | `location.href` at snapshot time |

Each AX node:

| Field | Type | Required | Description |
|---|---|---|---|
| `role` | string | yes | WAI-ARIA role (`button`, `link`, `heading`, `textbox`, `slider`, `checkbox`, `region`, etc.) |
| `accessibleName` | string | yes | Computed accessible name (from `aria-label`, `aria-labelledby`, visible text, or `<label>` association) |
| `value` | any | no | For form controls: current value |
| `state` | object | no | Subset of: `disabled`, `checked`, `expanded`, `pressed`, `selected`, `focused` (booleans) |
| `children` | array | no | Nested AX nodes |

The inbound `snapshot-request` event MAY include a `targetTab` field with the same semantics as `action.targetTab` and `configure.targetTab`. Absent → every shim emits a snapshot.

## type: action

Emitted by the backend toward the shim, requesting a synthetic interaction.

| Field | Type | Required | Description |
|---|---|---|---|
| `id` | string | yes | Unique action id (shim echoes in `action-result`) |
| `target` | object | yes | `{role, accessibleName, nth?}` — see I3 |
| `action` | string | yes | `click`, `input`, `keypress`, `focus`, `blur` |
| `value` | string | conditional | Required for `input` and `keypress` |
| `targetTab` | string | no | When set, only the shim whose `tabId` matches dispatches this action. Absent → broadcast. |

I3 contract: `target` has exactly these keys: `role` (required, non-empty string), `accessibleName` (required, non-empty string), `nth` (optional, non-negative integer). Any other key rejects the action.

## type: action-result

Emitted by the shim after executing (or rejecting) an action.

| Field | Type | Required | Description |
|---|---|---|---|
| `id` | string | yes | Matches the `id` on the originating action |
| `ok` | boolean | yes | Whether the action completed without error |
| `error` | string | conditional | Present iff `ok: false` — one of: `no-match`, `multiple-matches-need-nth`, `invalid-address`, `action-threw` |
| `resolvedRole` | string | no | Echo of the resolved element's role (aids debugging) |
| `resolvedAccessibleName` | string | no | Echo of the resolved element's accessible name |

## type: shim-ready

Emitted by the shim once per page mount, immediately after console proxy + SSE connection are established.

| Field | Type | Required | Description |
|---|---|---|---|
| `url` | string | yes | `location.href` at mount time |
| `shimVersion` | string | yes | Semver string of the adapted shim. Drivers use this to detect version mismatch and log compatibility warnings. |
| `tabId` | string | yes | UUID generated by the shim at mount. Persists across SSE reconnects within the same page; resets on navigation. Used by drivers to scope inbound events via `targetTab`. |

Example:
`{"type":"shim-ready","ts":"2026-04-23T19:30:00.000Z","url":"http://localhost:5173/design","shimVersion":"0.4.0","tabId":"7c4a3f81-9d2e-4b8f-a1c3-2d6e5f7a8b9c"}`

On page reload or navigation, the shim re-emits `shim-ready` with a NEW `tabId` (new mount = new UUID) and resets session state (notably `watch_dom` returns to empty). Drivers running a live capture session replay their `configure` event when a new `shim-ready` arrives, using the new `tabId` for any `targetTab`-scoped subsequent commands.

## type: configure

Emitted by the backend toward the shim to adjust session knobs.

| Field | Type | Required | Description |
|---|---|---|---|
| `watch_dom` | array of string | no | Array of event type names (subset of `click`, `submit`, `input`, `keydown`) — shim attaches capture-phase listeners for listed events and detaches any currently-attached event not in the list. Empty array detaches all. Absent key leaves existing state unchanged. |
| `targetTab` | string | no | When set, only the shim whose `tabId` matches handles this event. Absent → broadcast (every shim handles). |

Future session knobs become additional optional keys on the same event without breaking older shims. Absent key always means "no change" — preserves forward compatibility.

Example (v1.2 form):
`{"type":"configure","ts":"2026-04-23T19:30:01.000Z","watch_dom":["click","input"]}`


## type: dom-event

Emitted by the shim for each capture-phase event in the currently-watched set that reaches an AX-addressable target.

| Field | Type | Required | Description |
|---|---|---|---|
| `eventType` | string | yes | One of `click`, `submit`, `input`, `keydown`. Other values reserved for future versions. |
| `target` | object | yes | `{role, accessibleName}` — subset of the I3 action-address shape. `nth` is reserved for a future version. |
| `value` | string | conditional | Present only for `eventType: "input"` when the target's role is `textbox`, `searchbox`, or `combobox`. The post-input value of the field at capture time (DOM `input` event semantics). Omitted for all other events/roles. |
| `key` | string | conditional | Present only for `eventType: "keydown"`. The `event.key` value (e.g. `"Enter"`, `"a"`, `"Escape"`). No modifiers, no keyCode. |
| `ctrlKey` | boolean | yes (when `eventType === "keydown"`) | Whether the Ctrl key was held when the keydown fired |
| `metaKey` | boolean | yes (when `eventType === "keydown"`) | Whether the Meta/Cmd key was held |
| `altKey` | boolean | yes (when `eventType === "keydown"`) | Whether the Alt/Option key was held |
| `shiftKey` | boolean | yes (when `eventType === "keydown"`) | Whether the Shift key was held |

If the event target has no computed role or empty accessible name, the shim drops the event silently — consistent with I3: AX-unreachable elements are a11y bugs, not events to forward. No synthetic "unknown" values are ever produced.

Examples:

`{"type":"dom-event","ts":"2026-04-23T19:30:02.000Z","eventType":"click","target":{"role":"button","accessibleName":"Add text"}}`

`{"type":"dom-event","ts":"2026-04-23T19:30:02.100Z","eventType":"input","target":{"role":"textbox","accessibleName":"Caption"},"value":"hello world"}`

`{"type":"dom-event","ts":"2026-04-23T19:30:02.200Z","eventType":"keydown","target":{"role":"textbox","accessibleName":"Caption"},"key":"s","ctrlKey":false,"metaKey":true,"altKey":false,"shiftKey":false}`

## Transport

The protocol is transport-neutral. `install-bridge` picks per stack — SSE, WebSocket, or polling. The wire payload above is the contract.
